Security Lab

취약점 개념을 단순 암기가 아니라 재현, 원인 분석, 대응, 재검증 흐름으로 정리합니다.

STEP 01

SQL Injection

인증 우회, UNION 기반 조회, Blind SQL Injection을 실습하고 PreparedStatement 기반 방어 방식을 정리합니다.

STEP 02

XSS

Stored, Reflected, DOM XSS의 차이를 정리하고, 출력 인코딩, CSP, HttpOnly Cookie 적용 여부를 점검합니다.

STEP 03

IDOR & Access Control

사용자 ID, 게시글 ID, 댓글 ID를 조작했을 때 권한 검증이 정상적으로 동작하는지 확인합니다.

STEP 04

Path Traversal & File Upload

파일 다운로드, 업로드 기능에서 경로 조작, 확장자 우회, MIME 타입 조작 가능성을 점검합니다.
